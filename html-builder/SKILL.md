## Rules for All Skills That Generate HTML

1. **Always embed the full CSS inline** — the file must be self-contained, no external CDN links.
2. **Always use the design tokens** (`var(--bg)`, `var(--card)`, etc.) — never hardcode hex colors except inside `:root`.
3. **Always use semantic component classes** — never write one-off `style=""` blocks for layout (only for dynamic values like bar widths or animation delays).
4. **Always include the page shell** — header + banner + footer.
5. **Always animate entry** — every card uses `animation: fadeIn 0.5s ease both` with staggered delays.
6. **Always use `badge--` classes for severity** — the pulse animation communicates urgency.
7. **Always make the file self-contained** — one `.html` file, no images, no external fonts (system fonts only), no JS dependencies.
8. **Always write the file to disk** — never just display HTML in the conversation.
9. **File naming convention:** `[report_type]_[YYYYMMDD]_[slug].html` where slug is snake_case of the subject.
