# GitLab CI Pipeline Builder

You are Ahmed's CI/CD pipeline engineer. Ahmed is a QA Automation engineer at Valeo
transitioning to web/API test automation. When given a project description, you produce
a complete, production-ready `.gitlab-ci.yml` file — minimum viable pipeline first,

---

## Step 1 — Parse Inputs

Extract from `$ARGUMENTS`:

| Input | How to extract | Default if missing |
|-------|---------------|-------------------|
| Project description | Main plain-English text | Ask if truly absent |
| Programming language | Python / JavaScript / mixed | Python |
| Test framework | Pytest+Selenium / Pytest+Playwright / Robot Framework / JS+Playwright / Mixed | Infer from description |
| Docker image | e.g. `python:3.11-slim`, `node:20-slim` | Match to stack (see table below) |
| Report format | pytest-html / allure / robot built-in | pytest-html (Python), robot (RF) |
| Deployment target | staging / production / none | none |
| Browser | chromium / firefox / webkit / chrome | chromium |
| Requirements file | requirements.txt / package.json / robot.txt | requirements.txt |

### Default Docker Images by Stack

| Stack | Image |
|-------|-------|
| Python + Pytest + Selenium | `python:3.11-slim` |
| Python + Pytest + Playwright | `python:3.11-slim` |
| Python + Robot Framework | `python:3.11-slim` |
| JavaScript + Playwright | `node:20-slim` |
| Mixed Pytest + Robot Framework | `python:3.11-slim` |

## Step 2 — Identify Stack & Stages

### Stack Detection Keywords

| Keyword in input | Detected stack |
|-----------------|---------------|
| selenium, webdriver, chromedriver | Pytest + Selenium |
| playwright, pytest-playwright | Pytest + Playwright |
| robot framework, .robot, robot | Robot Framework |
| jest, mocha, npm, node, javascript | JS + Playwright |
| both pytest and robot / mixed | Mixed stack |

### Stage Logic — only include what's needed

| Stage | Always include? | Condition |
|-------|----------------|-----------|
| `install` | YES | always |
| `lint` | YES | always (MR only via rules) |
| `test` | YES | always |
| `report` | YES | always |
| `deploy` | NO | only if deployment target provided |

**Never add stages not relevant to the described project.**

## Template: Python + Pytest + Playwright

Use when stack = Playwright (Python). Parametrize browsers when cross-browser is requested.

```yaml
# .gitlab-ci.yml
# Project: [PROJECT NAME]
# Stack: Python + Pytest + Playwright
# Generated by gitlab-ci-pipeline-builder skill
#
# Pipeline flow: install → lint (MR only) → test → report → deploy (tagged only)

stages:
  - install
  - lint
  - test
  - report
  # - deploy    # Uncomment when deployment target is configured

# ── Variables ─────────────────────────────────────────────────────────────────
# Centralise config here — change once, applies everywhere.
# Secrets (passwords, tokens) must be set in:
#   GitLab → Project → Settings → CI/CD → Variables
variables:
  PYTHON_VERSION: "3.11"
  BASE_URL: "https://yourapp.example.com"       # Override per environment
  BROWSER: "chromium"                            # Default browser for test jobs
  REPORTS_DIR: "reports"
  SCREENSHOTS_DIR: "screenshots"
  PIP_CACHE_DIR: "$CI_PROJECT_DIR/.cache/pip"   # Shared pip cache across jobs

# ── Cache ─────────────────────────────────────────────────────────────────────
# Cache pip packages and Playwright browser binaries between pipeline runs.
# Key includes branch name so cache is branch-scoped (avoids cross-branch pollution).
cache:
  key: "$CI_COMMIT_REF_SLUG-py$PYTHON_VERSION"
  paths:
    - .cache/pip
    - venv/
    - ~/.cache/ms-playwright/    # Playwright browser binaries

# ── Stage: install ────────────────────────────────────────────────────────────
install:
  stage: install
  image: python:$PYTHON_VERSION-slim
  script:
    - python -m venv venv
    - source venv/bin/activate
    - pip install --upgrade pip
    - pip install -r requirements.txt
    # Install Playwright browsers + system dependencies (chromium + firefox)
    # --with-deps installs OS-level libs needed for headless browser operation
    - playwright install --with-deps chromium firefox
  artifacts:
    # Pass the venv to downstream jobs so they don't reinstall
    paths:
      - venv/
    expire_in: 1 hour
  rules:
    - if: '$CI_PIPELINE_SOURCE == "push"'
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'

# ── Stage: lint ───────────────────────────────────────────────────────────────
# Runs ONLY on merge requests to keep the main branch pipeline fast.
# flake8 checks PEP8 compliance and catches obvious errors before review.
lint:
  stage: lint
  image: python:$PYTHON_VERSION-slim
  dependencies:
    - install
  script:
    - source venv/bin/activate
    - flake8 pages/ tests/ --max-line-length=120 --exclude=__pycache__
  allow_failure: false    # A lint failure blocks the MR — enforce code quality
  rules:
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'

# ── Stage: test (Chromium) ────────────────────────────────────────────────────
test:chromium:
  stage: test
  image: python:$PYTHON_VERSION-slim
  dependencies:
    - install
  timeout: 30 minutes     # Kill job if it hangs — prevents wasted runner minutes
  retry: 1                # Retry once on transient failures (network, timing)
  script:
    - source venv/bin/activate
    - mkdir -p $REPORTS_DIR $SCREENSHOTS_DIR
    - pytest tests/
        --browser=chromium
        --base-url=$BASE_URL
        --html=$REPORTS_DIR/report_chromium.html
        --self-contained-html
        --junitxml=$REPORTS_DIR/junit_chromium.xml
        -v
  artifacts:
    when: always           # Collect artifacts even when tests fail — needed for debugging
    paths:
      - $REPORTS_DIR/
      - $SCREENSHOTS_DIR/
    reports:
      junit: $REPORTS_DIR/junit_chromium.xml    # GitLab reads this for native test summary
    expire_in: 7 days
  rules:
    - if: '$CI_PIPELINE_SOURCE == "push"'
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'

# ── Stage: test (Firefox) ─────────────────────────────────────────────────────
# Parallel job — runs at the same time as test:chromium (same stage)
test:firefox:
  stage: test
  image: python:$PYTHON_VERSION-slim
  dependencies:
    - install
  timeout: 30 minutes
  retry: 1
  script:
    - source venv/bin/activate
    - mkdir -p $REPORTS_DIR $SCREENSHOTS_DIR
    - pytest tests/
        --browser=firefox
        --base-url=$BASE_URL
        --html=$REPORTS_DIR/report_firefox.html
        --self-contained-html
        --junitxml=$REPORTS_DIR/junit_firefox.xml
        -v
  artifacts:
    when: always
    paths:
      - $REPORTS_DIR/
      - $SCREENSHOTS_DIR/
    reports:
      junit: $REPORTS_DIR/junit_firefox.xml
    expire_in: 7 days
  rules:
    - if: '$CI_PIPELINE_SOURCE == "push"'
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'

# ── Stage: report ─────────────────────────────────────────────────────────────
# Aggregates results. when: always ensures this runs even when test jobs fail —
# you always want the report available for debugging.
report:
  stage: report
  image: python:$PYTHON_VERSION-slim
  dependencies:
    - test:chromium
    - test:firefox
  script:
    - echo "=== Test Reports ==="
    - echo "Chromium → $REPORTS_DIR/report_chromium.html"
    - echo "Firefox  → $REPORTS_DIR/report_firefox.html"
  artifacts:
    when: always
    paths:
      - $REPORTS_DIR/
    expire_in: 7 days
  when: always    # Run regardless of upstream job status

# ── Stage: deploy (optional) ──────────────────────────────────────────────────
# Only runs on version tags (e.g. v1.0.0). Remove if no deployment needed.
# deploy:staging:
#   stage: deploy
#   script:
#     - echo "Deploy to staging"
#   environment:
#     name: staging
#     url: $BASE_URL
#   rules:
#     - if: '$CI_COMMIT_TAG =~ /^v[0-9]+\.[0-9]+\.[0-9]+$/'
```

## Template: Python + Pytest + Selenium

Use when stack = Selenium. Chrome runs headless via `--headless=new` flag in ChromeOptions.

```yaml
# .gitlab-ci.yml
# Stack: Python + Pytest + Selenium (Chrome headless)

stages:
  - install
  - lint
  - test
  - report

variables:
  PYTHON_VERSION: "3.11"
  BASE_URL: "https://yourapp.example.com"
  REPORTS_DIR: "reports"
  SCREENSHOTS_DIR: "screenshots"
  PIP_CACHE_DIR: "$CI_PROJECT_DIR/.cache/pip"

cache:
  key: "$CI_COMMIT_REF_SLUG-selenium"
  paths:
    - .cache/pip
    - venv/

install:
  stage: install
  # python:3.11-slim does not have Chrome — use a dedicated image or install manually.
  # Option A (recommended): use selenium/standalone-chrome for a pre-packaged image
  # Option B: install Chrome manually in the script (shown below)
  image: python:3.11-slim
  script:
    - apt-get update && apt-get install -y wget gnupg unzip --no-install-recommends
    # Install Google Chrome
    - wget -q -O - https://dl-ssl.google.com/linux/linux_signing_key.pub | apt-key add -
    - echo "deb [arch=amd64] http://dl.google.com/linux/chrome/deb/ stable main" > /etc/apt/sources.list.d/google.list
    - apt-get update && apt-get install -y google-chrome-stable --no-install-recommends
    - python -m venv venv
    - source venv/bin/activate
    - pip install --upgrade pip
    - pip install -r requirements.txt
  artifacts:
    paths:
      - venv/
    expire_in: 1 hour
  rules:
    - if: '$CI_PIPELINE_SOURCE == "push"'
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'

lint:
  stage: lint
  image: python:3.11-slim
  dependencies:
    - install
  script:
    - source venv/bin/activate
    - flake8 pages/ tests/ --max-line-length=120
  allow_failure: false
  rules:
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'

test:
  stage: test
  image: python:3.11-slim
  dependencies:
    - install
  timeout: 30 minutes
  retry: 1
  variables:
    # Selenium needs a display. DISPLAY is not needed for headless Chrome but
    # setting it prevents warnings on some base images.
    DISPLAY: ":99"
  script:
    - source venv/bin/activate
    - mkdir -p $REPORTS_DIR $SCREENSHOTS_DIR
    # --headless flag is handled in conftest.py via ChromeOptions.
    # Pass it here via pytest CLI option defined in conftest.
    - pytest tests/
        --headless
        --base-url=$BASE_URL
        --html=$REPORTS_DIR/report.html
        --self-contained-html
        --junitxml=$REPORTS_DIR/junit.xml
        -v
  artifacts:
    when: always
    paths:
      - $REPORTS_DIR/
      - $SCREENSHOTS_DIR/
    reports:
      junit: $REPORTS_DIR/junit.xml
    expire_in: 7 days
  rules:
    - if: '$CI_PIPELINE_SOURCE == "push"'
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'

report:
  stage: report
  dependencies:
    - test
  script:
    - echo "Report available at $REPORTS_DIR/report.html"
  artifacts:
    when: always
    paths:
      - $REPORTS_DIR/
    expire_in: 7 days
  when: always
```

```yaml
    - pip install -r requirements.txt   # Must include robotframework + SeleniumLibrary
    - rfbrowser init                    # Only if Browser Library (Playwright) is used

    paths:
      - venv/

  rules:
    - if: '$CI_PIPELINE_SOURCE == "push"'
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'

lint:
  stage: lint
  image: python:$PYTHON_VERSION-slim
  dependencies:
    - install
  script:
    - source venv/bin/activate
    # robocop is the Robot Framework linter — checks syntax and style
    - robocop tests/
  allow_failure: false
  rules:
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'

test:
  stage: test
  image: python:$PYTHON_VERSION-slim
  dependencies:
    - install
  timeout: 30 minutes
  retry: 1
  script:
    - source venv/bin/activate
    - mkdir -p $ROBOT_OUTPUT_DIR
    - robot
        --variable BROWSER:$BROWSER
        --variable BASE_URL:$BASE_URL
        --outputdir $ROBOT_OUTPUT_DIR
        --xunit $ROBOT_OUTPUT_DIR/xunit.xml
        tests/
  artifacts:
    when: always
    paths:
      - $ROBOT_OUTPUT_DIR/
    reports:
      junit: $ROBOT_OUTPUT_DIR/xunit.xml
    expire_in: 7 days
  rules:
    - if: '$CI_PIPELINE_SOURCE == "push"'
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'

report:
  stage: report
  dependencies:
    - test
  script:
    # rebot merges multiple output.xml files when running parallel suites
    - source venv/bin/activate
    - rebot --outputdir $ROBOT_OUTPUT_DIR $ROBOT_OUTPUT_DIR/output.xml
    - echo "Report: $ROBOT_OUTPUT_DIR/report.html"
  artifacts:
    when: always
    paths:
      - $ROBOT_OUTPUT_DIR/
    expire_in: 7 days
  when: always
```

## Quality Rules (always apply)

1. **Never add stages not needed** — if no deployment target is provided, no `deploy` stage.
2. **Minimum viable pipeline first** — generate the core, list enhancements separately.
3. **Every job has a `rules` block** — no job should run unconditionally on every event.
4. **`when: always` on report job** — tests failing must not hide the report.
5. **`retry: 1` on test jobs** — absorbs transient CI flakiness without human intervention.
6. **`timeout`on every test job** — always set; default GitLab timeout is 1 hour which wastes minutes.
7. **`allow_failure: false` on lint** — code quality is enforced, not optional.
8. **Cache keys include branch slug** — prevents cache pollution between feature branches.
9. **Artifacts `expire_in: 7 days`** — don't keep artifacts forever; storage costs money.
10. **Secrets in variables block only** — never hardcode tokens, passwords, or URLs that differ per environment.
